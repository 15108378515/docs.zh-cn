# <a name="compiler-warning-level-2-cs0467"></a><span data-ttu-id="83e48-102">编译器警告（等级 2）CS0467</span><span class="sxs-lookup"><span data-stu-id="83e48-102">Compiler Warning (level 2) CS0467</span></span>
<span data-ttu-id="83e48-103">方法“method”和非方法“non-method”之间存在多义性。使用方法组。</span><span class="sxs-lookup"><span data-stu-id="83e48-103">Ambiguity between method 'method' and non-method 'non-method'.Using method group.</span></span>  
  
 <span data-ttu-id="83e48-104">如果继承的成员具有相同的签名，但来自不同的接口，则会导致多义性错误。</span><span class="sxs-lookup"><span data-stu-id="83e48-104">Inherited members from different interfaces that have the same signature  cause an ambiguity error.</span></span>  
  
## <a name="example"></a><span data-ttu-id="83e48-105">示例</span><span class="sxs-lookup"><span data-stu-id="83e48-105">Example</span></span>  
 <span data-ttu-id="83e48-106">下面的示例生成 CS0467。</span><span class="sxs-lookup"><span data-stu-id="83e48-106">The following example generates CS0467.</span></span>  
  
```csharp  
// CS0467.cs  
interface IList   
{  
    int Count { get; set; }  
}  
  
interface ICounter  
{  
    void Count(int i);  
}  
  
interface IListCounter : IList, ICounter {}  
  
class Driver   
{  
    void Test(IListCounter x)  
    {  
        // The following line causes the warning. The assignment also  
        // causes an error because you can't assign a value to a method.  
        x.Count = 1;  
        x.Count(3);     
        // To resolve the warning, you can change the name of the method or   
        // the property.  
  
        // You also can disambiguate by specifying IList or ICounter.  
        ((IList)x).Count = 1;  
        ((ICounter)x).Count(3);  
    }  
  
    static void Main()   
    {  
    }  
}  
```
