<span data-ttu-id="b2c1e-103">访问器“accessor”不能为“type”类型实现接口成员“member”。</span><span class="sxs-lookup"><span data-stu-id="b2c1e-103">Accessor 'accessor' cannot implement interface member 'member' for type 'type'.</span></span> <span data-ttu-id="b2c1e-104">使用显式接口实现。</span><span class="sxs-lookup"><span data-stu-id="b2c1e-104">Use an explicit interface implementation.</span></span>  
  
 <span data-ttu-id="b2c1e-105">建议：实现一个接口时，如果其中包含的方法名称和与属性或事件关联的自动生成的方法相冲突，会出现此错误。</span><span class="sxs-lookup"><span data-stu-id="b2c1e-105">Suggested: This error can occur when implementing an interface that contains method names which conflict with the auto-generated methods associated with a property or event.</span></span> <span data-ttu-id="b2c1e-106">属性的 get/set 方法生成为 get_property 和 set_property，事件的 add/remove 方法生成为 add_event 和 remove_event。</span><span class="sxs-lookup"><span data-stu-id="b2c1e-106">The get/set methods for properties are generated as get_property and set_property, and the add/remove methods for events are generated as add_event and remove_event.</span></span> <span data-ttu-id="b2c1e-107">如果接口包含以上任一方法，将发生冲突。</span><span class="sxs-lookup"><span data-stu-id="b2c1e-107">If an interface contains either of these methods, a conflict occurs.</span></span> <span data-ttu-id="b2c1e-108">若要避免此错误，请使用显式接口实现来实现这些方法。</span><span class="sxs-lookup"><span data-stu-id="b2c1e-108">To avoid this error, implement the methods using an explicit interface implementation.</span></span> <span data-ttu-id="b2c1e-109">若要执行此操作，将函数指定为：</span><span class="sxs-lookup"><span data-stu-id="b2c1e-109">To do this, specify the function as:</span></span>

  
## <a name="example"></a><span data-ttu-id="b2c1e-110">示例</span><span class="sxs-lookup"><span data-stu-id="b2c1e-110">Example</span></span>  
 <span data-ttu-id="b2c1e-111">下面的示例生成 CS0686：</span><span class="sxs-lookup"><span data-stu-id="b2c1e-111">The following sample generates CS0686:</span></span>  
  
```csharp  
// CS0686.cs  
interface I  
{  
    int get_P();  
}  
  
class C : I  
{  
    public int P  
    {  
        get { return 1; }  // CS0686  
    }  
}  
// But the following is valid:  
class D : I  
{  
    int I.get_P() { return 1; }  
    public static void Main() {}  
}  
```  
  
## <a name="example"></a><span data-ttu-id="b2c1e-112">示例</span><span class="sxs-lookup"><span data-stu-id="b2c1e-112">Example</span></span>  
 <span data-ttu-id="b2c1e-113">声明事件时，也会发生此错误。</span><span class="sxs-lookup"><span data-stu-id="b2c1e-113">This error can also occur when declaring events.</span></span>  <span data-ttu-id="b2c1e-114">事件构造自动生成 `add_``event` 和 `remove_``event` 方法，这两个方法会与接口中同名的方法发生冲突，如以下示例所示：</span><span class="sxs-lookup"><span data-stu-id="b2c1e-114">The event construct automatically generates the `add_``event` and `remove_``event` methods, which could conflict with the methods of the same name in an interface, as in the following sample:</span></span>  
  
```csharp  
// CS0686b.cs  
using System;  
  
interface I  
{  
    void add_OnMyEvent(EventHandler e);  
}  
  
class C : I  
{  
    public event EventHandler OnMyEvent  
    {  
        add { }  // CS0686  
        remove { }  
    }  
}  
  
// Correct (using explicit interface implementation):  
class D : I  
{  
    void I.add_OnMyEvent(EventHandler e) {}  
    public static void Main() {}  
}  
```
