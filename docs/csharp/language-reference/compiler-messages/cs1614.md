# <a name="compiler-error-cs1614"></a><span data-ttu-id="6a56a-102">编译器错误 CS1614</span><span class="sxs-lookup"><span data-stu-id="6a56a-102">Compiler Error CS1614</span></span>
<span data-ttu-id="6a56a-103">“name”不明确；“attribute1”和“attribute2”之间。</span><span class="sxs-lookup"><span data-stu-id="6a56a-103">'name' is ambiguous; between 'attribute1' and 'attribute2'.</span></span> <span data-ttu-id="6a56a-104">使用“@attribute”或“attributeAttribute”</span><span class="sxs-lookup"><span data-stu-id="6a56a-104">use either '@attribute' or 'attributeAttribute'</span></span>  
  
 <span data-ttu-id="6a56a-105">编译器遇到了不明确的属性指定。</span><span class="sxs-lookup"><span data-stu-id="6a56a-105">The compiler has encountered an ambiguous attribute specification.</span></span>  
  
 <span data-ttu-id="6a56a-106">为方便起见，C# 编译器允许将“ExampleAttribute”指定为 `[Example]`。</span><span class="sxs-lookup"><span data-stu-id="6a56a-106">For convenience, the C# compiler allows you to specify **ExampleAttribute** as just `[Example]`.</span></span> <span data-ttu-id="6a56a-107">但是，如果名为 `Example` 的属性类与“ExampleAttribute”同时存在，则会引起歧义，因为编译器无法确定 `[Example]` 是指 `Example` 属性还是“ExampleAttribute”属性。</span><span class="sxs-lookup"><span data-stu-id="6a56a-107">However, ambiguity arises if an attribute class named `Example` exists along with **ExampleAttribute**, because the compiler cannot tell if `[Example]` refers to the `Example` attribute or the **ExampleAttribute** attribute.</span></span> <span data-ttu-id="6a56a-108">为阐释清楚，将 `[@Example]` 用作 `Example` 属性，将 `[ExampleAttribute]` 用作“ExampleAttribute”。</span><span class="sxs-lookup"><span data-stu-id="6a56a-108">To clarify, use `[@Example]` for the `Example` attribute and `[ExampleAttribute]` for **ExampleAttribute**.</span></span>  
  
 <span data-ttu-id="6a56a-109">下面的示例生成 CS1614：</span><span class="sxs-lookup"><span data-stu-id="6a56a-109">The following sample generates CS1614:</span></span>  
  
```csharp  
// CS1614.cs  
using System;  
  
// Both of the following classes are valid attributes with valid  
// names (MySpecial and MySpecialAttribute). However, because the lookup  
// rules for attributes involves auto-appending the 'Attribute' suffix  
// to the identifier, these two attributes become ambiguous; that is,  
// if you specify MySpecial, the compiler can't tell if you want  
// MySpecial or MySpecialAttribute.  
  
public class MySpecial : Attribute {  
   public MySpecial() {}  
}  
  
public class MySpecialAttribute : Attribute {  
   public MySpecialAttribute() {}  
}  
  
class MakeAWarning {  
   [MySpecial()] // CS1614  
                 // Ambiguous: MySpecial or MySpecialAttribute?  
   public static void Main() {  
   }  
  
   [@MySpecial()] // This isn't ambiguous, it binds to the first attribute above.  
   public static void NoWarning() {  
   }  
  
   [MySpecialAttribute()] // This isn't ambiguous, it binds to the second attribute above.  
   public static void NoWarning2() {  
   }  
  
   [@MySpecialAttribute()] // This is also legal.  
   public static void NoWarning3() {  
   }  
}  
```
