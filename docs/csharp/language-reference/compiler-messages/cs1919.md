# <a name="compiler-error-cs1919"></a><span data-ttu-id="914d3-102">编译器错误 CS1919</span><span class="sxs-lookup"><span data-stu-id="914d3-102">Compiler Error CS1919</span></span>
<span data-ttu-id="914d3-103">创建对象时不能使用不安全的类型“type name”。</span><span class="sxs-lookup"><span data-stu-id="914d3-103">Unsafe type 'type name' cannot be used in object creation.</span></span>  
  
 <span data-ttu-id="914d3-104">`new` 运算符仅在托管堆上创建对象。</span><span class="sxs-lookup"><span data-stu-id="914d3-104">The `new` operator creates objects only on the managed heap.</span></span> <span data-ttu-id="914d3-105">但可以通过使用语言的互操作性功能调用返回指针的本机方法，间接地在非托管内存中创建对象。</span><span class="sxs-lookup"><span data-stu-id="914d3-105">However, you can create objects in unmanaged memory indirectly by using the interoperability capabilities of the language to call native methods that return pointers.</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="914d3-106">更正此错误</span><span class="sxs-lookup"><span data-stu-id="914d3-106">To correct this error</span></span>  
  
1.  <span data-ttu-id="914d3-107">在新的对象创建表达式中使用安全类型。</span><span class="sxs-lookup"><span data-stu-id="914d3-107">Use a safe type in the new object creation expression.</span></span> <span data-ttu-id="914d3-108">例如，使用 `char` 或 `int`，而不是使用 `char*` 或 `int*`。</span><span class="sxs-lookup"><span data-stu-id="914d3-108">For example, use `char` or `int` instead of `char*` or `int*`.</span></span>  
  
2.  <span data-ttu-id="914d3-109">如果必须在非托管内存中创建对象，请使用 Win32 或 COM 方法，或者用 C 或 C++ 编写自己的函数并从 C# 调用该函数。</span><span class="sxs-lookup"><span data-stu-id="914d3-109">If you must create objects in unmanaged memory, use a Win32 or COM method or else write your own function in C or C++ and call it from C#.</span></span>  
  
## <a name="example"></a><span data-ttu-id="914d3-110">示例</span><span class="sxs-lookup"><span data-stu-id="914d3-110">Example</span></span>  
 <span data-ttu-id="914d3-111">下面的示例生成 CS1919，因为指针类型是不安全的：</span><span class="sxs-lookup"><span data-stu-id="914d3-111">The following example generates CS1919 because a pointer type is unsafe:</span></span>  
  
```csharp  
// cs1919.cs  
// Compile with: /unsafe  
unsafe public class C  
{  
    public static int Main()  
    {  
        var col1 = new int* { }; // CS1919  
        var col2 = new char* { }; // CS1919  
        return 1;  
    }  
}  
```
