# <a name="compiler-warning-level-1-cs3007"></a><span data-ttu-id="bc3a5-102">编译器警告（等级 1）CS3007</span><span class="sxs-lookup"><span data-stu-id="bc3a5-102">Compiler Warning (level 1) CS3007</span></span>
<span data-ttu-id="bc3a5-103">只是未命名数组类型不同的重载方法“method”不符合 CLS。</span><span class="sxs-lookup"><span data-stu-id="bc3a5-103">Overloaded method 'method' differing only by unnamed array types is not CLS-compliant</span></span>  
  
 <span data-ttu-id="bc3a5-104">如果你有一个接受交错数组的重载方法，并且方法签名的唯一区别是数组的元素类型，则将发生此错误。</span><span class="sxs-lookup"><span data-stu-id="bc3a5-104">This error occurs if you have an overloaded method that takes a jagged array and the only difference between the method signatures is the element type of the array.</span></span> <span data-ttu-id="bc3a5-105">为避免此错误，可考虑使用矩形数组而不是交错数组，使用一个额外的参数消除函数调用的歧义，重命名一个或多个重载方法，或者如果不需要 CLS 遵从性，则移除 <xref:System.CLSCompliantAttribute> 特性。</span><span class="sxs-lookup"><span data-stu-id="bc3a5-105">To avoid this error, consider using a rectangular array rather than a jagged array; use an additional parameter to disambiguate the function call; rename one or more of the overloaded methods; or, if CLS Compliance is not needed, remove the <xref:System.CLSCompliantAttribute> attribute.</span></span> <span data-ttu-id="bc3a5-106">有关 CLS 符合性的详细信息，请参阅[语言独立性和与语言无关的组件](https://msdn.microsoft.com/library/12a7a7h3)。</span><span class="sxs-lookup"><span data-stu-id="bc3a5-106">For more information on CLS Compliance, see [Language Independence and Language-Independent Components](https://msdn.microsoft.com/library/12a7a7h3).</span></span>  
  
## <a name="example"></a><span data-ttu-id="bc3a5-107">示例</span><span class="sxs-lookup"><span data-stu-id="bc3a5-107">Example</span></span>  
 <span data-ttu-id="bc3a5-108">下面的示例生成 CS3007：</span><span class="sxs-lookup"><span data-stu-id="bc3a5-108">The following example generates CS3007:</span></span>  
  
```csharp  
// CS3007.cs  
[assembly: System.CLSCompliant(true)]  
public struct S  
{  
    public void F(int[][] array) { }  
    public void F(byte[][] array) { }  // CS3007  
    // Try this instead:  
    // public void F1(int[][] array) {}  
    // public void F2(byte[][] array) {}  
    // or   
    // public void F(int[,] array) {}  
    // public void F(byte[,] array) {}  
}  
```
