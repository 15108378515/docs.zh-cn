# <a name="compiler-warning-level-2-cs0440"></a><span data-ttu-id="e71d4-102">编译器警告（等级 2）CS0440</span><span class="sxs-lookup"><span data-stu-id="e71d4-102">Compiler Warning (level 2) CS0440</span></span>
<span data-ttu-id="e71d4-103">由于 "global::" 总是引用全局命名空间而非别名，因此定义一个名为 "global" 的别名是欠妥的</span><span class="sxs-lookup"><span data-stu-id="e71d4-103">Defining an alias named 'global' is ill-advised since 'global::' always references the global namespace and not an alias</span></span>  
  
 <span data-ttu-id="e71d4-104">在定义名为 global 的别名时会发出此警告。</span><span class="sxs-lookup"><span data-stu-id="e71d4-104">This warning is issued when you define an alias named global.</span></span>  
  
## <a name="example"></a><span data-ttu-id="e71d4-105">示例</span><span class="sxs-lookup"><span data-stu-id="e71d4-105">Example</span></span>  
 <span data-ttu-id="e71d4-106">下面的示例生成 CS0440：</span><span class="sxs-lookup"><span data-stu-id="e71d4-106">The following example generates CS0440:</span></span>  
  
```csharp  
// CS0440.cs  
// Compile with: /W:1  
  
using global = MyClass;   // CS0440  
class MyClass  
{  
    static void Main()  
    {  
        // Note how global refers to the global namespace  
        // even though it is redefined above.  
        global::System.Console.WriteLine();  
    }  
}  
```
