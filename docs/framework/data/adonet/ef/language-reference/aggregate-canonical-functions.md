## <a name="group-based-aggregates"></a><span data-ttu-id="1d328-182">基于组的聚合</span><span class="sxs-lookup"><span data-stu-id="1d328-182">Group-based aggregates</span></span>

<span data-ttu-id="1d328-183">基于组的聚合将按照 GROUP BY 子句定义的方式对组进行计算。</span><span class="sxs-lookup"><span data-stu-id="1d328-183">Group-based aggregates are calculated over a group as defined by the GROUP BY clause.</span></span> <span data-ttu-id="1d328-184">对于结果中的每个组，将使用每个组中的元素作为聚合计算的输入来计算单独的聚合。</span><span class="sxs-lookup"><span data-stu-id="1d328-184">For each group in the result, a separate aggregate is calculated by using the elements in each group as input to the aggregate calculation.</span></span> <span data-ttu-id="1d328-185">当在 select 表达式中使用 group-by 子句时，在投影或 order-by 子句中只存在分组表达式名称、聚合或常量表达式。</span><span class="sxs-lookup"><span data-stu-id="1d328-185">When a group-by clause is used in a select expression, only grouping expression names, aggregates, or constant expressions can be present in the projection or order-by clause.</span></span>

<span data-ttu-id="1d328-186">以下示例计算每种产品的平均订购数量：</span><span class="sxs-lookup"><span data-stu-id="1d328-186">The following example calculates the average quantity ordered for each product:</span></span>

```sql
select p, avg(ol.Quantity) from LOB.OrderLines as ol 
  group by ol.Product as p
```

<span data-ttu-id="1d328-187">它是可以在 SELECT 表达式中包含没有显式 group-by 子句基于组的聚合。</span><span class="sxs-lookup"><span data-stu-id="1d328-187">It's possible to have a group-based aggregate without an explicit group-by clause in the SELECT expression.</span></span> <span data-ttu-id="1d328-188">在这种情况下，所有元素都视为单个组。</span><span class="sxs-lookup"><span data-stu-id="1d328-188">In this case, all elements are treated as a single group.</span></span> <span data-ttu-id="1d328-189">这是等效的指定基于常数的分组。</span><span class="sxs-lookup"><span data-stu-id="1d328-189">This is equivalent of specifying a grouping based on a constant.</span></span> <span data-ttu-id="1d328-190">例如，请看下面的表达式：</span><span class="sxs-lookup"><span data-stu-id="1d328-190">Take, for example, the following expression:</span></span>

```sql
select avg(ol.Quantity) from LOB.OrderLines as ol
```

<span data-ttu-id="1d328-191">此表达式等效于以下表达式：</span><span class="sxs-lookup"><span data-stu-id="1d328-191">This is equivalent to the following:</span></span>

```sql
select avg(ol.Quantity) from LOB.OrderLines as ol group by 1
```

<span data-ttu-id="1d328-193">作为 in [!INCLUDE[tsql](../../../../../../includes/tsql-md.md)]，基于组的聚合也可以指定 ALL 或 DISTINCT 修饰符。</span><span class="sxs-lookup"><span data-stu-id="1d328-193">As in [!INCLUDE[tsql](../../../../../../includes/tsql-md.md)], group-based aggregates can also specify an ALL or DISTINCT modifier.</span></span> <span data-ttu-id="1d328-194">如果指定 DISTINCT 修饰符，则将从聚合输入集合中消除重复项，然后计算聚合。</span><span class="sxs-lookup"><span data-stu-id="1d328-194">If the DISTINCT modifier is specified, duplicates are eliminated from the aggregate input collection, before the aggregate is computed.</span></span> <span data-ttu-id="1d328-195">如果指定 ALL 修饰符（或者未指定修饰符），则不执行重复项消除。</span><span class="sxs-lookup"><span data-stu-id="1d328-195">If the ALL modifier is specified (or if no modifier is specified), no duplicate elimination is performed.</span></span>
