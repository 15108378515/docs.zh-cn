<span data-ttu-id="67af6-103">COM 方法通过返回 HRESULT 来报告错误；NET 方法通过引发异常来报告错误。</span><span class="sxs-lookup"><span data-stu-id="67af6-103">COM methods report errors by returning HRESULTs; .NET methods report them by throwing exceptions.</span></span> <span data-ttu-id="67af6-104">运行时处理这两者之间的转换。</span><span class="sxs-lookup"><span data-stu-id="67af6-104">The runtime handles the transition between the two.</span></span> <span data-ttu-id="67af6-105">NET Framework 中的每个异常类都将映射到 HRESULT。</span><span class="sxs-lookup"><span data-stu-id="67af6-105">Each exception class in the .NET Framework maps to an HRESULT.</span></span>  
  
 <span data-ttu-id="67af6-106">用户定义的异常类可指定任何适用的 HRESULT。</span><span class="sxs-lookup"><span data-stu-id="67af6-106">User-defined exception classes can specify whatever HRESULT is appropriate.</span></span> <span data-ttu-id="67af6-107">这些异常类可动态更改通过设置异常对象上的 HResult 字段生成异常时返回的 HRESULT。</span><span class="sxs-lookup"><span data-stu-id="67af6-107">These exception classes can dynamically change the HRESULT to be returned when the exception is generated by setting the **HResult** field on the exception object.</span></span> <span data-ttu-id="67af6-108">异常的其他信息通过 IErrorInfo 接口提供给客户端，这是在非管理进程中的 NET 对象上实现的。</span><span class="sxs-lookup"><span data-stu-id="67af6-108">Additional information about the exception is provided to the client through the **IErrorInfo** interface, which is implemented on the .NET object in the unmanaged process.</span></span>  
  
 <span data-ttu-id="67af6-109">如果要创建扩展 System.Exception 的类，需要在构造时设置 HRESULT 字段。</span><span class="sxs-lookup"><span data-stu-id="67af6-109">If you create a class that extends **System.Exception**, you must set the HRESULT field during construction.</span></span> <span data-ttu-id="67af6-110">否则，基类将分配 HRESULT 值。</span><span class="sxs-lookup"><span data-stu-id="67af6-110">Otherwise, the base class assigns the HRESULT value.</span></span> <span data-ttu-id="67af6-111">可以通过提供异常的构造函数的值，将新的异常类映射到现有的 HRESULT。</span><span class="sxs-lookup"><span data-stu-id="67af6-111">You can map new exception classes to an existing HRESULT by supplying the value in the exception's constructor.</span></span>

### <a name="to-create-a-new-exception-class-and-map-it-to-an-hresult"></a><span data-ttu-id="67af6-114">创建新的异常类并将其映射到 HRESULT</span><span class="sxs-lookup"><span data-stu-id="67af6-114">To create a new exception class and map it to an HRESULT</span></span>  
  
1.  <span data-ttu-id="67af6-115">使用以下代码创建名为 `NoAccessException` 的新的异常类，并将其映射到 HRESULT `E_ACCESSDENIED`。</span><span class="sxs-lookup"><span data-stu-id="67af6-115">Use the following code to create a new exception class called `NoAccessException` and map it to the HRESULT `E_ACCESSDENIED`.</span></span>

    ```cpp  
    {  
        NoAccessException () {  
        HResult = E_ACCESSDENIED;

    }

    {  
    throw new NoAccessException();  
    }  
    ```
