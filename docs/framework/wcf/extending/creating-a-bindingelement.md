## <a name="adding-a-binding-element"></a><span data-ttu-id="9818a-110">添加绑定元素</span><span class="sxs-lookup"><span data-stu-id="9818a-110">Adding a Binding Element</span></span>

 <span data-ttu-id="9818a-116"><xref:System.ServiceModel.Channels.BindingElement.BuildChannelFactory%2A> 首先检查是否可以生成请求的通道形状，然后获取要分割的消息操作列表。</span><span class="sxs-lookup"><span data-stu-id="9818a-116"><xref:System.ServiceModel.Channels.BindingElement.BuildChannelFactory%2A> first checks that the requested channel shape can be built and then gets a list of message actions to be chunked.</span></span> <span data-ttu-id="9818a-117">然后，它创建一个新的 `ChunkingChannelFactory`，并为其传递内部通道工厂。</span><span class="sxs-lookup"><span data-stu-id="9818a-117">It then creates a new `ChunkingChannelFactory`, passing it the inner channel factory.</span></span> <span data-ttu-id="9818a-118">（如果您要创建传输绑定元素，该元素将是绑定堆栈中的最后一个元素，因此必须创建一个通道侦听器或通道工厂。）</span><span class="sxs-lookup"><span data-stu-id="9818a-118">(If you are creating a transport binding element, that element is the last one in the binding stack and therefore must create a channel listener or channel factory.)</span></span>  
  
 <span data-ttu-id="9818a-119"><xref:System.ServiceModel.Channels.BindingElement.BuildChannelListener%2A> 有一个类似的实现，用于创建 `ChunkingChannelListener` 并为其传递内部通道侦听器。</span><span class="sxs-lookup"><span data-stu-id="9818a-119"><xref:System.ServiceModel.Channels.BindingElement.BuildChannelListener%2A> has a similar implementation for creating `ChunkingChannelListener` and passing it the inner channel listener.</span></span>  
  
 <span data-ttu-id="9818a-120">另一个示例是使用传输通道，[传输： UDP](../../../../docs/framework/wcf/samples/transport-udp.md)示例提供了以下替代。</span><span class="sxs-lookup"><span data-stu-id="9818a-120">As another example using a transport channel, the [Transport: UDP](../../../../docs/framework/wcf/samples/transport-udp.md) sample provides the following override.</span></span>
