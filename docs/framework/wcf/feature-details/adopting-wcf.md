# <a name="adopting-windows-communication-foundation"></a><span data-ttu-id="59f26-102">采用 Windows Communication Foundation</span><span class="sxs-lookup"><span data-stu-id="59f26-102">Adopting Windows Communication Foundation</span></span>
<span data-ttu-id="59f26-103">你可以选择要用于新开发，Windows Communication Foundation (WCF) 时使用 ASP.NET 继续保持现有应用程序开发。</span><span class="sxs-lookup"><span data-stu-id="59f26-103">You can choose to use Windows Communication Foundation (WCF) for new development, while continuing to maintain existing applications developed using ASP.NET.</span></span> <span data-ttu-id="59f26-104">WCF 用于生成与任何方案中的.NET Framework 的应用程序之间通信最适当的选择，因为它可以充当以用于解决各种软件通信问题的方式的标准工具该 ASP.NET不能。</span><span class="sxs-lookup"><span data-stu-id="59f26-104">Because WCF is intended to be the most suitable choice for facilitating communication with applications built with the .NET Framework in any scenario, it can serve as a standard tool for solving a wide variety of software communications problems in a way that ASP.NET cannot.</span></span>  
  
 <span data-ttu-id="59f26-105">可以在作为现有的 ASP.NET Web 服务相同的计算机上部署新的 WCF 应用程序。</span><span class="sxs-lookup"><span data-stu-id="59f26-105">New WCF applications can be deployed on the same machines as existing ASP.NET Web services.</span></span> <span data-ttu-id="59f26-106">如果现有的 ASP.NET Web 服务使用的.NET Framework 2.0 版之前的版本，然后可以使用 ASP.NET IIS 注册工具有选择地将.NET Framework 2.0 部署到新的 WCF 应用程序要承载的 IIS 应用程序。</span><span class="sxs-lookup"><span data-stu-id="59f26-106">If the existing ASP.NET Web services use a version of the .NET Framework prior to version 2.0, then you can use the ASP.NET IIS Registration Tool to selectively deploy the .NET Framework 2.0 to IIS applications in which new WCF applications are to be hosted.</span></span> <span data-ttu-id="59f26-107">该工具记录在[ASP.NET IIS 注册工具 (Aspnet_regiis.exe)](http://go.microsoft.com/fwlink/?LinkId=94687)，并具有用户界面内置于 IIS 6.0 管理控制台。</span><span class="sxs-lookup"><span data-stu-id="59f26-107">That tool is documented at [ASP.NET IIS Registration Tool (Aspnet_regiis.exe)](http://go.microsoft.com/fwlink/?LinkId=94687), and has a user interface built into the IIS 6.0 management console.</span></span>  
  
 <span data-ttu-id="59f26-108">WCF 可用来将新功能添加到现有的 ASP.NET Web 服务，通过添加配置为对在 IIS 中的现有 ASP.NET Web 服务应用程序在 ASP.NET 兼容模式中运行的 WCF 服务。</span><span class="sxs-lookup"><span data-stu-id="59f26-108">WCF can be used to add new features to existing ASP.NET Web services by adding WCF services configured to run in ASP.NET compatibility mode to existing ASP.NET Web service applications in IIS.</span></span> <span data-ttu-id="59f26-109">由于 ASP.NET 兼容模式，新的 WCF 服务的代码可以访问和使用中更新预先存在的 ASP.NET 代码，与相同的应用程序状态信息<xref:System.Web.HttpContext>类。</span><span class="sxs-lookup"><span data-stu-id="59f26-109">Because of ASP.NET compatibility mode, the code for the new WCF services can access and update the same application state information as the pre-existing ASP.NET code, by using the <xref:System.Web.HttpContext> class.</span></span> <span data-ttu-id="59f26-110">这些应用程序还可以共享相同的类库。</span><span class="sxs-lookup"><span data-stu-id="59f26-110">The applications can also share the same class libraries.</span></span>  
  
 <span data-ttu-id="59f26-111">WCF 客户端可以使用 ASP.NET Web 服务。</span><span class="sxs-lookup"><span data-stu-id="59f26-111">WCF clients can use ASP.NET Web services.</span></span> <span data-ttu-id="59f26-112">使用配置的 WCF 服务<xref:System.ServiceModel.BasicHttpBinding>可由 ASP.NET Web 服务客户端。</span><span class="sxs-lookup"><span data-stu-id="59f26-112">WCF services that are configured with the <xref:System.ServiceModel.BasicHttpBinding> can be used by ASP.NET Web service clients.</span></span> <span data-ttu-id="59f26-113">ASP.NET Web 服务可以共存与 WCF 应用程序，并甚至可以使用 WCF 将功能添加到现有的 ASP.NET Web 服务。</span><span class="sxs-lookup"><span data-stu-id="59f26-113">ASP.NET Web services can co-exist with WCF applications, and WCF can even be used to add features to existing ASP.NET Web services.</span></span> <span data-ttu-id="59f26-114">有了所有的 WCF 和 ASP.NET Web 服务可以一起使用这些方法，你可能想要将 ASP.NET Web 服务迁移到 WCF，只有当你需要通过 WCF 和不是 ASP.NET Web 服务提供的功能。</span><span class="sxs-lookup"><span data-stu-id="59f26-114">Given all of these ways in which WCF and ASP.NET Web services can be used together, you may want to migrate ASP.NET Web services to WCF only if you require features that are provided by WCF and not ASP.NET Web services.</span></span>  
  
 <span data-ttu-id="59f26-115">即使在需要进行迁移的少数几种情况下，也请仔细考虑一下，因为将代码从一种技术迁移到另一种技术很少成为一种正确的方法。</span><span class="sxs-lookup"><span data-stu-id="59f26-115">Even in the few cases where it is necessary, carefully consider that migrating code from one technology to another is seldom the correct approach.</span></span> <span data-ttu-id="59f26-116">采用新技术的原因是为了满足早期技术无法满足的新需求。在上述情况下，正确的做法是设计一个新的解决方案来满足刚刚扩展的需求集。</span><span class="sxs-lookup"><span data-stu-id="59f26-116">The reason for adopting the new technology is to meet new requirements that cannot be met with the earlier technology, and in that case, the correct thing to do is to design a new solution to meet the newly-expanded set of requirements.</span></span> <span data-ttu-id="59f26-117">新的设计将受益于您对现有系统所拥有的经验以及您自该系统被设计以来获得的智慧。</span><span class="sxs-lookup"><span data-stu-id="59f26-117">The new design benefits from your experience with the existing system and from wisdom gained since that system was designed.</span></span> <span data-ttu-id="59f26-118">此外，新设计还可以利用新技术的所有功能，而不是在新平台上再现旧设计。</span><span class="sxs-lookup"><span data-stu-id="59f26-118">The new design can also use the full capabilities of the new technologies rather than reproducing the old design on the new platform.</span></span> <span data-ttu-id="59f26-119">在建立新设计的关键元素的原型之后，可以更轻松地在新系统中重用现有系统中的代码。</span><span class="sxs-lookup"><span data-stu-id="59f26-119">After prototyping key elements of the new design, it becomes easier to re-use code from the existing system within the new one.</span></span>  
  
 <span data-ttu-id="59f26-120">少数情况下从 ASP.NET Web 服务迁移到 WCF 的位置是正确的解决方案下, 一节将提供有关如何继续的一些指导。</span><span class="sxs-lookup"><span data-stu-id="59f26-120">For the few cases where porting from ASP.NET Web services to WCF is the correct solution, the following section provides some guidance on how to proceed.</span></span> <span data-ttu-id="59f26-121">其中就如何迁移服务以及如何迁移客户端提出了建议。</span><span class="sxs-lookup"><span data-stu-id="59f26-121">There is advice on how to migrate services, and how to migrate clients.</span></span>  
  
 <span data-ttu-id="59f26-122">有关如何将现有的 ASP.NET Web 服务迁移到 WCF 的完整分析，请参阅[ASP.NET Web 服务和 Windows Communication Foundation](http://go.microsoft.com/fwlink/?LinkID=71761)。</span><span class="sxs-lookup"><span data-stu-id="59f26-122">For a complete analysis on how to migrate existing ASP.NET Web Services to WCF please see [ASP.NET Web Services and the Windows Communication Foundation](http://go.microsoft.com/fwlink/?LinkID=71761).</span></span> <span data-ttu-id="59f26-123">本部分介绍如何为你的 ASP.NET Web 服务，实现兼容 WCF 服务的元数据以及如何将 ASP.NET Web 服务和客户端代码迁移到 WCF。</span><span class="sxs-lookup"><span data-stu-id="59f26-123">This section describes how to implement a compliant WCF service from the metadata for you ASP.NET Web Service, and how to migrate ASP.NET Web service and client code to WCF.</span></span>
