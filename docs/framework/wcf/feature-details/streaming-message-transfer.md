# <a name="streaming-message-transfer"></a><span data-ttu-id="b3a21-102">流消息传输</span><span class="sxs-lookup"><span data-stu-id="b3a21-102">Streaming Message Transfer</span></span>
[!INCLUDE[indigo1](../../../../includes/indigo1-md.md)]<span data-ttu-id="b3a21-103"> 传输机制支持两种消息传输模式：</span><span class="sxs-lookup"><span data-stu-id="b3a21-103"> transports support two modes for transferring messages:</span></span>  
  
-   <span data-ttu-id="b3a21-104">缓冲传输将整个消息保留在内存缓冲区中，直到传输完成。</span><span class="sxs-lookup"><span data-stu-id="b3a21-104">Buffered transfers hold the entire message in a memory buffer until the transfer is complete.</span></span> <span data-ttu-id="b3a21-105">必须完整传递缓冲的消息，接收方才能读取该消息。</span><span class="sxs-lookup"><span data-stu-id="b3a21-105">A buffered message must be completely delivered before a receiver can read it.</span></span>  
  
-   <span data-ttu-id="b3a21-106">流传输以流的形式公开消息。</span><span class="sxs-lookup"><span data-stu-id="b3a21-106">Streamed transfers expose the message as a stream.</span></span> <span data-ttu-id="b3a21-107">接收方在消息完整传递之前即可开始处理消息。</span><span class="sxs-lookup"><span data-stu-id="b3a21-107">The receiver starts processing the message before it is completely delivered.</span></span>  
  
-   <span data-ttu-id="b3a21-108">流传输消除了对大型内存缓冲区的要求，从而提高了服务的可伸缩性。</span><span class="sxs-lookup"><span data-stu-id="b3a21-108">Streamed transfers can improve the scalability of a service by eliminating the requirement for large memory buffers.</span></span> <span data-ttu-id="b3a21-109">更改传输模式是否能够提高可伸缩性取决于所传输的消息大小。</span><span class="sxs-lookup"><span data-stu-id="b3a21-109">Whether changing the transfer mode improves scalability depends on the size of the messages being transferred.</span></span> <span data-ttu-id="b3a21-110">消息大小越大，使用流传输越有利。</span><span class="sxs-lookup"><span data-stu-id="b3a21-110">Large message sizes favor using streamed transfers.</span></span>  
  
 <span data-ttu-id="b3a21-111">默认情况下，HTTP、TCP/IP 和命名管道传输协议使用缓冲传输。</span><span class="sxs-lookup"><span data-stu-id="b3a21-111">By default, the HTTP, TCP/IP, and named pipe transports use buffered transfers.</span></span> <span data-ttu-id="b3a21-112">本文档介绍如何将这些传输协议从缓冲传输模式切换到流传输模式以及这样做的结果。</span><span class="sxs-lookup"><span data-stu-id="b3a21-112">This document describes how to switch these transports from a buffered to streamed transfer mode and the consequences of doing so.</span></span>

 <span data-ttu-id="b3a21-118"><xref:System.ServiceModel.BasicHttpBinding>、<xref:System.ServiceModel.NetTcpBinding> 和 <xref:System.ServiceModel.NetNamedPipeBinding> 绑定公开了 <xref:System.ServiceModel.TransferMode> 属性。</span><span class="sxs-lookup"><span data-stu-id="b3a21-118">The <xref:System.ServiceModel.BasicHttpBinding>, <xref:System.ServiceModel.NetTcpBinding>, and <xref:System.ServiceModel.NetNamedPipeBinding> bindings expose the <xref:System.ServiceModel.TransferMode> property.</span></span> <span data-ttu-id="b3a21-119">对于其他传输，必须创建一个自定义绑定以设置传输模式。</span><span class="sxs-lookup"><span data-stu-id="b3a21-119">For other transports, you must create a custom binding to set the transfer mode.</span></span>

## <a name="enabling-asynchronous-streaming"></a><span data-ttu-id="b3a21-126">启用异步流处理</span><span class="sxs-lookup"><span data-stu-id="b3a21-126">Enabling Asynchronous Streaming</span></span>

 <span data-ttu-id="b3a21-128">此版本的 WCF 还在发送端添加真正异步流处理的能力。</span><span class="sxs-lookup"><span data-stu-id="b3a21-128">This version of WCF also adde the capability of true asynchronous streaming on the send side.</span></span> <span data-ttu-id="b3a21-129">在将消息流式处理到多个客户端的方案中，这可提高服务的可扩展性；其中某些客户端的读取速度较慢，这可能由于网络拥塞或根本无法阅读造成的。</span><span class="sxs-lookup"><span data-stu-id="b3a21-129">This improves scalability of the service in scenarios where it is streaming messages to multiple clients some of which are slow in reading; possibly due to network congestion or are not reading at all.</span></span> <span data-ttu-id="b3a21-130">在这些方案中，WCF 不再因每个客户端的服务阻止各个线程。</span><span class="sxs-lookup"><span data-stu-id="b3a21-130">In these scenarios WCF no longer blocks individual threads on the service per client.</span></span> <span data-ttu-id="b3a21-131">这确保服务能够处理多得多的客户端，进而提高服务的可扩展性。</span><span class="sxs-lookup"><span data-stu-id="b3a21-131">This ensures that the service is able to process many more clients thereby improving the scalability of the service.</span></span>
