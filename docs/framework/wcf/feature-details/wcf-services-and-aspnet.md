 <span data-ttu-id="33c29-137">尽管 [!INCLUDE[indigo2](../../../../includes/indigo2-md.md)] 模型旨在跨宿主环境和传输保持行为的一致性，但经常在一些方案中，应用程序中不要求这种程度的灵活性。</span><span class="sxs-lookup"><span data-stu-id="33c29-137">Although the [!INCLUDE[indigo2](../../../../includes/indigo2-md.md)] model is designed to behave consistently across hosting environments and transports, there are often scenarios where an application does not require this degree of flexibility.</span></span> [!INCLUDE[indigo2](../../../../includes/indigo2-md.md)]<span data-ttu-id="33c29-138"> 的 ASP.NET 兼容模式适用于具有以下特点的方案：不需要具有在 IIS 外部承载或通过 HTTP 之外的协议进行通信的能力，但使用 ASP.NET Web 应用程序平台的所有功能。</span><span class="sxs-lookup"><span data-stu-id="33c29-138">’s ASP.NET compatibility mode is suitable for scenarios that do not require the ability to host outside of IIS or to communicate over protocols other than HTTP, but that use all of features of the ASP.NET Web application platform.</span></span>
