 <span data-ttu-id="8f003-105">本示例包含一个服务以及向该服务进行请求的客户端代码。</span><span class="sxs-lookup"><span data-stu-id="8f003-105">The sample consists of a service along with client code that makes requests to the service.</span></span> <span data-ttu-id="8f003-106">该服务支持一个 HTTP `GET` 操作 (`EchoWithGet`) 和一个 HTTP `POST` 操作 (`EchoWithPost`)。</span><span class="sxs-lookup"><span data-stu-id="8f003-106">The service supports a single HTTP `GET` operation (`EchoWithGet`) and a single HTTP `POST` operation (`EchoWithPost`).</span></span> <span data-ttu-id="8f003-107">这两个操作都需要一个字符串，然后在响应中返回字符串。</span><span class="sxs-lookup"><span data-stu-id="8f003-107">Both operations expect a string and then return the string in the response.</span></span> <span data-ttu-id="8f003-108">对于 `GET` 操作，字符串在 URI 查询字符串参数中提供。</span><span class="sxs-lookup"><span data-stu-id="8f003-108">With the `GET` operation, the string is provided in a URI query-string parameter.</span></span> <span data-ttu-id="8f003-109">对于 `POST` 操作，字符串在请求正文中提供（以 XML 格式序列化）。</span><span class="sxs-lookup"><span data-stu-id="8f003-109">With the `POST` operation, the string is provided in the body of the request, serialized in XML.</span></span> <span data-ttu-id="8f003-110">利用 [!INCLUDE[netfx40_long](../../../../includes/netfx40-long-md.md)] 中新的自动格式选择和命令性格式选择功能，服务能够以 XML 或 JSON 格式返回响应。</span><span class="sxs-lookup"><span data-stu-id="8f003-110">The service is able to return responses in either XML or JSON, utilizing the new automatic format selection and imperative format selection features in [!INCLUDE[netfx40_long](../../../../includes/netfx40-long-md.md)].</span></span>  
  
 <span data-ttu-id="8f003-111">在本示例中，使用 App.config 文件启用自动格式选择。</span><span class="sxs-lookup"><span data-stu-id="8f003-111">In the sample, automatic format selection is enabled using the App.config file.</span></span> <span data-ttu-id="8f003-112">在默认的 Web HTTP 终结点上，为 `automaticFormatSelectionEnabled` 特性提供值 `true`。</span><span class="sxs-lookup"><span data-stu-id="8f003-112">On the default Web HTTP endpoint, the `automaticFormatSelectionEnabled` attribute has been given a value of `true`.</span></span> <span data-ttu-id="8f003-113">启用自动格式选择，WCF 基础结构会选择最适合的响应格式 （XML 或 JSON） 给定请求的 HTTP Accept 或 Content-type 标头。</span><span class="sxs-lookup"><span data-stu-id="8f003-113">With automatic format selection enabled, the WCF infrastructure selects the most appropriate response format (XML or JSON) given the HTTP Accept or Content-Type headers of the request.</span></span> <span data-ttu-id="8f003-114">除了将 `automaticFormatSelectionEnabled` 特性设置为 `true`，开发人员不需要提供任何其他代码或配置便可使用此新功能。</span><span class="sxs-lookup"><span data-stu-id="8f003-114">The developer is not required to provide any additional code or configuration other than setting the `automaticFormatSelectionEnabled` attribute to `true` to use this new feature.</span></span> <span data-ttu-id="8f003-115">在 Program.cs 中的客户端代码，请求会发送对这两个`GET`和`POST`的 HTTP Accept 标头指定为"application/xml"或"application/json"的服务和服务操作返回的响应中的相应的格式。</span><span class="sxs-lookup"><span data-stu-id="8f003-115">In the client code in Program.cs, requests are sent to both the `GET` and `POST` operations of the service with the HTTP Accept header specified as either "application/xml" or "application/json" and the service returns a response in that respective format.</span></span>  
  
 <span data-ttu-id="8f003-116">在 `GET` 操作中，还使用了命令性格式选择。</span><span class="sxs-lookup"><span data-stu-id="8f003-116">Also in the `GET` operation, imperative format selection is used.</span></span> <span data-ttu-id="8f003-117">`GET` 操作检查是否存在可选的 `format` 查询字符串参数，如果存在，则在 <xref:System.ServiceModel.Web.WebOperationContext.OutgoingResponse%2A> 属性上设置响应格式。</span><span class="sxs-lookup"><span data-stu-id="8f003-117">The `GET` operation checks for an optional `format` query-string parameter and if present, sets the response format on the <xref:System.ServiceModel.Web.WebOperationContext.OutgoingResponse%2A> property.</span></span> <span data-ttu-id="8f003-118">强制设置响应格式以这种方式重写完成 WCF 基础结构的自动格式选择。</span><span class="sxs-lookup"><span data-stu-id="8f003-118">Imperatively setting the response format in this way overrides the automatic format selection done by the WCF infrastructure.</span></span>  
  
 <span data-ttu-id="8f003-119">此示例包含一个自承载服务和一个客户端，它们都在一个控制台应用程序内运行。</span><span class="sxs-lookup"><span data-stu-id="8f003-119">The sample consists of a self-hosted service and a client that runs within a console application.</span></span> <span data-ttu-id="8f003-120">在控制台应用程序运行时，客户端会对服务进行请求，并将响应中的相关信息写入控制台窗口。</span><span class="sxs-lookup"><span data-stu-id="8f003-120">As the console application runs, the client makes requests to the service and writes the pertinent information from the responses to the console window.</span></span>
