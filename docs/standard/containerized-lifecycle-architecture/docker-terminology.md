# <a name="docker-terminology"></a><span data-ttu-id="56450-103">Docker 术语</span><span class="sxs-lookup"><span data-stu-id="56450-103">Docker terminology</span></span>

<span data-ttu-id="56450-104">本部分列出术语和定义与其应与你熟悉之前深入了解更深入地 Docker (有关进一步定义，请参阅对各种[术语表](https://docs.docker.com/glossary/)由在 Docker <https://docs.docker.com/glossary/>:</span><span class="sxs-lookup"><span data-stu-id="56450-104">This section lists terms and definitions with which you should become familiar with before delving deeper into Docker (for further definitions, see the extensive [glossary](https://docs.docker.com/glossary/) provided by Docker at <https://docs.docker.com/glossary/>:</span></span>

-   <span data-ttu-id="56450-105">**容器映像** 的所有依赖关系和创建容器所需信息的包。</span><span class="sxs-lookup"><span data-stu-id="56450-105">**Container image** A package with all of the dependencies and information needed to create a container.</span></span> <span data-ttu-id="56450-106">映像包括所有依赖项 （如框架） 以及部署和使用由容器运行时配置。</span><span class="sxs-lookup"><span data-stu-id="56450-106">An image includes all of the dependencies (such as frameworks) plus deployment and configuration to be used by a container runtime.</span></span> <span data-ttu-id="56450-107">通常情况下，映像派生自多个基映像，层将堆积一个之上打开另一个用于窗体的容器的文件系统。</span><span class="sxs-lookup"><span data-stu-id="56450-107">Usually, an image derives from multiple base images that are layers stacked one atop the other to form the container's file system.</span></span> <span data-ttu-id="56450-108">它已被创建后，映像是不可变。</span><span class="sxs-lookup"><span data-stu-id="56450-108">An image is immutable after it has been created.</span></span>

-   <span data-ttu-id="56450-109">**容器** Docker 映像的实例。</span><span class="sxs-lookup"><span data-stu-id="56450-109">**Container** An instance of a Docker image.</span></span> <span data-ttu-id="56450-110">容器表示为单个应用程序、 进程或服务的运行时。</span><span class="sxs-lookup"><span data-stu-id="56450-110">A container represents a runtime for a single application, process, or service.</span></span> <span data-ttu-id="56450-111">它包含的内容的 Docker 映像、 运行时环境和一组标准的说明。</span><span class="sxs-lookup"><span data-stu-id="56450-111">It consists of the contents of a Docker image, a runtime environment, and a standard set of instructions.</span></span> <span data-ttu-id="56450-112">在缩放服务时，可以从相同的映像创建多个容器实例。</span><span class="sxs-lookup"><span data-stu-id="56450-112">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="56450-113">或者，批处理作业可以从同一个映像，将不同的参数传递给每个实例创建多个容器。</span><span class="sxs-lookup"><span data-stu-id="56450-113">Or, a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

-   <span data-ttu-id="56450-114">**标记** 标记或标签，以便可以确定不同的映像或相同的映像 （具体取决于的版本号或目标环境） 的版本，可以应用于映像。</span><span class="sxs-lookup"><span data-stu-id="56450-114">**Tag** A mark or label that you can apply to images so that different images or versions of the same image (depending on the version number or the destination environment) can be identified.</span></span>

-   <span data-ttu-id="56450-115">**Dockerfile** 文本文件，其中包含有关如何生成的 Docker 映像的说明。</span><span class="sxs-lookup"><span data-stu-id="56450-115">**Dockerfile** A text file that contains instructions for how to build a Docker image.</span></span>

-   <span data-ttu-id="56450-116">**生成** 上的信息和上下文由其 Dockerfile，以及在其中生成映像的文件夹中的其他文件基于生成的容器映像的操作。</span><span class="sxs-lookup"><span data-stu-id="56450-116">**Build** The action of building a container image based on the information and context provided by its Dockerfile as well as additional files in the folder where the image is built.</span></span> <span data-ttu-id="56450-117">你可以通过使用 Docker docker 生成命令生成映像。</span><span class="sxs-lookup"><span data-stu-id="56450-117">You can build images by using the Docker docker build command.</span></span>

-   <span data-ttu-id="56450-118">**存储库 （也称为存储库）** 相关的标记，该值指示映像版本标记为的 Docker 映像的集合。</span><span class="sxs-lookup"><span data-stu-id="56450-118">**Repository (also known as repo)** A collection of related Docker images labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="56450-119">某些存储库包含多个不同版本的特定映像，如包含映像包含仅运行时 （较亮） 的 Sdk （更大） 的映像，依此类推。</span><span class="sxs-lookup"><span data-stu-id="56450-119">Some repositories contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), and so on.</span></span> <span data-ttu-id="56450-120">这些变量可以使用标记进行标记。</span><span class="sxs-lookup"><span data-stu-id="56450-120">Those variants can be marked with tags.</span></span> <span data-ttu-id="56450-121">单个存储库可以包含平台的不同版本，如 Linux 映像和 Windows 映像。</span><span class="sxs-lookup"><span data-stu-id="56450-121">A single repository can contain platform variants, such as a Linux image and a Windows image.</span></span>

-   <span data-ttu-id="56450-122">**注册表** 提供对存储库的访问的服务。</span><span class="sxs-lookup"><span data-stu-id="56450-122">**Registry** A service that provides access to repositories.</span></span> <span data-ttu-id="56450-123">大多数公共映像的默认注册表是[Docker Hub](https://hub.docker.com/)（归作为组织的 Docker 所有）。</span><span class="sxs-lookup"><span data-stu-id="56450-123">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="56450-124">注册表通常包含来自多个团队的存储库。</span><span class="sxs-lookup"><span data-stu-id="56450-124">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="56450-125">通常，公司具有私有注册表来存储和管理他们创建的映像。</span><span class="sxs-lookup"><span data-stu-id="56450-125">Companies often have private registries to store and manage images that they've created.</span></span> <span data-ttu-id="56450-126">*Azure 容器注册表*是另一个示例。</span><span class="sxs-lookup"><span data-stu-id="56450-126">*Azure Container Registry* is another example.</span></span>

-   <span data-ttu-id="56450-127">**Docker Hub** 公共注册表将图像上载和使用它们。</span><span class="sxs-lookup"><span data-stu-id="56450-127">**Docker Hub** A public registry to upload images and work with them.</span></span> <span data-ttu-id="56450-128">Docker Hub 提供 Docker 映像托管、公共或私有注册表，生成触发器和 Web 挂钩，以及与 GitHub 和 Bitbucket 集成。</span><span class="sxs-lookup"><span data-stu-id="56450-128">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

-   <span data-ttu-id="56450-129">**Azure 容器注册表** 使用 Docker 映像和在 Azure 中的及其组件的公共资源。</span><span class="sxs-lookup"><span data-stu-id="56450-129">**Azure Container Registry** A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="56450-130">这提供了接近 Azure 中部署的注册表，授予控制访问权限，使其可以使用 Azure Active Directory 组和权限。</span><span class="sxs-lookup"><span data-stu-id="56450-130">This provides a registry that is close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

-   <span data-ttu-id="56450-131">**Docker 受信任注册表 (DTR)** 可以安装的 Docker 注册表服务 （从 Docker) 本地，以便它位于内组织的数据中心和网络。</span><span class="sxs-lookup"><span data-stu-id="56450-131">**Docker Trusted Registry (DTR)** A Docker registry service (from Docker) that you can install on-premises so that it resides within the organization's datacenter and network.</span></span> <span data-ttu-id="56450-132">这对于应该在企业内部管理的私有映像来说很方便。</span><span class="sxs-lookup"><span data-stu-id="56450-132">It is convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="56450-133">Docker 受信任注册表是 Docker 数据中心产品的一部分。</span><span class="sxs-lookup"><span data-stu-id="56450-133">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="56450-134">有关详细信息，请转到[ https://docs.docker.com/docker-trusted-registry/overview/ ](https://docs.docker.com/docker-trusted-registry/overview/)。</span><span class="sxs-lookup"><span data-stu-id="56450-134">For more information, go to [https://docs.docker.com/docker-trusted-registry/overview/](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

-   <span data-ttu-id="56450-135">**Docker 社区版 (CE)** 适用于 Windows 和 macOS 有关生成、 运行和测试容器本地的开发工具。</span><span class="sxs-lookup"><span data-stu-id="56450-135">**Docker Community Edition (CE)** Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="56450-136">适用于 Windows 的 Docker CE 为 Linux 和 Windows 容器提供了开发环境。</span><span class="sxs-lookup"><span data-stu-id="56450-136">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="56450-137">在 Windows 上的 Linux Docker 主机基于[HYPER-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) VM。</span><span class="sxs-lookup"><span data-stu-id="56450-137">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) VM.</span></span> <span data-ttu-id="56450-138">适用于 Windows 容器的主机直接基于 Windows。</span><span class="sxs-lookup"><span data-stu-id="56450-138">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="56450-139">适用于 Mac 的 docker CE 基于 Apple 虚拟机监控程序 framework 和[xhyve 虚拟机监控程序](https://github.com/mist64/xhyve)，提供了 Linux Docker 宿主 VM 上 Mac OS X.CE 用于 Windows 的 Docker 和适用于 Mac 替换 Docker 工具箱中，基于 Oracle VirtualBox。</span><span class="sxs-lookup"><span data-stu-id="56450-139">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host VM on Mac OS X. Docker CE for Windows and for Mac replaces Docker Toolbox, which was based on Oracle VirtualBox.</span></span>

-   <span data-ttu-id="56450-140">**Docker Enterprise Edition (EE)** 企业级版本的 Linux 和 Windows 开发的 Docker 工具。</span><span class="sxs-lookup"><span data-stu-id="56450-140">**Docker Enterprise Edition (EE)** An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

-   <span data-ttu-id="56450-141">**撰写** 命令行工具和 YAML 文件元数据的定义和运行 multicontainer 应用程序的格式。</span><span class="sxs-lookup"><span data-stu-id="56450-141">**Compose** A command-line tool and YAML file format with metadata for defining and running multicontainer applications.</span></span> <span data-ttu-id="56450-142">基于多个映像定义单个应用程序，其中包含一个或多个 .yml 文件，这些文件可以根据环境替代值。</span><span class="sxs-lookup"><span data-stu-id="56450-142">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="56450-143">你已创建定义后，你可以使用单个命令部署整个 multicontainer 应用程序 (docker-向上撰写) 在 Docker 主机上创建每个图像的容器。</span><span class="sxs-lookup"><span data-stu-id="56450-143">After you have created the definitions, you can deploy the entire multicontainer application by using a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>

-   <span data-ttu-id="56450-144">**群集** 分布在群集中的多个主机的 Docker 主机公开就像它们是一台虚拟 Docker 主机，以便应用程序可以扩展到服务的多个实例的集合。</span><span class="sxs-lookup"><span data-stu-id="56450-144">**Cluster** A collection of Docker hosts exposed as if they were a single virtual Docker host so that the application can scale to multiple instances of the services spread across multiple hosts within the cluster.</span></span> <span data-ttu-id="56450-145">你可以通过使用 Docker Swarm、 Mesosphere DC/OS、 Kubernetes 和 Azure Service Fabric 创建 Docker 群集。</span><span class="sxs-lookup"><span data-stu-id="56450-145">You can create Docker clusters by using Docker Swarm, Mesosphere DC/OS, Kubernetes, and Azure Service Fabric.</span></span> <span data-ttu-id="56450-146">（如果使用 Docker Swarm 来管理群集，通常是指将群集作为 swarm 而不是群集。）</span><span class="sxs-lookup"><span data-stu-id="56450-146">(If you use Docker Swarm for managing a cluster, you typically refer to the cluster as a *swarm* instead of a cluster.)</span></span>

-   <span data-ttu-id="56450-147">**Orchestrator** 简化了群集和 Docker 主机的管理工具。</span><span class="sxs-lookup"><span data-stu-id="56450-147">**Orchestrator** A tool that simplifies management of clusters and Docker hosts.</span></span> <span data-ttu-id="56450-148">使用 orchestrators，可以管理其映像、 容器和主机通过 CLI 或图形用户界面。</span><span class="sxs-lookup"><span data-stu-id="56450-148">Using orchestrators, you can manage their images, containers, and hosts through a CLI or a graphical user interface.</span></span> <span data-ttu-id="56450-149">可以管理容器网络、配置、负载均衡、服务发现、高可用性、Docker 主机配置等。</span><span class="sxs-lookup"><span data-stu-id="56450-149">You can manage container networking, configurations, load balancing, service discovery, high availability, Docker host configuration, and more.</span></span> <span data-ttu-id="56450-150">业务流程协调程序负责跨节点集合运行、分发、缩放和修复工作负荷。</span><span class="sxs-lookup"><span data-stu-id="56450-150">An orchestrator is responsible for running, distributing, scaling, and healing workloads across a collection of nodes.</span></span> <span data-ttu-id="56450-151">通常情况下，业务流程协调程序产品是提供群集基础结构的同一产品，如 Mesosphere DC/OS、Kubernetes、Docker Swarm 和 Azure Service Fabric。</span><span class="sxs-lookup"><span data-stu-id="56450-151">Typically, orchestrator products are the same products that provide cluster infrastructure, like Mesosphere DC/OS, Kubernetes, Docker Swarm, and Azure Service Fabric.</span></span>
