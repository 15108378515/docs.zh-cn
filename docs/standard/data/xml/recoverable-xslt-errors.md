# <a name="recoverable-xslt-errors"></a><span data-ttu-id="24bd1-102">可恢复的 XSLT 错误</span><span class="sxs-lookup"><span data-stu-id="24bd1-102">Recoverable XSLT Errors</span></span>
<span data-ttu-id="24bd1-103">W3C XSL 转换 (XSLT) 1.0 版建议中涉及到实现提供者可以在哪些方面确定如何处理某种情况。</span><span class="sxs-lookup"><span data-stu-id="24bd1-103">The W3C XSL Transformations (XSLT) Version 1.0 Recommendation includes areas in which the implementation provider may decide how to handle a situation.</span></span> <span data-ttu-id="24bd1-104">这些方面被认为是任意行为。</span><span class="sxs-lookup"><span data-stu-id="24bd1-104">These areas are considered to be discretionary behavior.</span></span> <span data-ttu-id="24bd1-105">例如，在第 7.3 节“Creating Processing Instructions”中，XSLT 1.0 建议指出，如果实例化 `xsl:processing-instruction` 的内容会创建文本节点以外的节点，则会发生错误。</span><span class="sxs-lookup"><span data-stu-id="24bd1-105">For example, in section 7.3 Creating Processing Instructions, the XSLT 1.0 recommendation states that it is an error if instantiating the content of `xsl:processing-instruction` creates nodes other than text nodes.</span></span> <span data-ttu-id="24bd1-106">对于某些问题，XSLT 1.0 建议指示在处理器决定从错误中恢复时应做的决策。</span><span class="sxs-lookup"><span data-stu-id="24bd1-106">For some problems, the XSLT 1.0 recommendation indicates what decision should be made if the processor decides to recover from the error.</span></span> <span data-ttu-id="24bd1-107">对于 7.3 节中给出的问题，W3C 指出，实现可以通过忽略节点及其内容来从此错误中恢复。</span><span class="sxs-lookup"><span data-stu-id="24bd1-107">For the problem given in section 7.3, the W3C says that the implementation can recover from this error by ignoring the nodes and their content.</span></span>
