<span data-ttu-id="7a546-107">**容器映像**： 所有依赖关系和创建容器所需信息的包。</span><span class="sxs-lookup"><span data-stu-id="7a546-107">**Container image**: A package with all the dependencies and information needed to create a container.</span></span> <span data-ttu-id="7a546-108">映像加上由容器运行时要使用部署和执行配置包括所有依赖项 （如框架）。</span><span class="sxs-lookup"><span data-stu-id="7a546-108">An image includes all the dependencies (such as frameworks) plus deployment and execution configuration to be used by a container runtime.</span></span> <span data-ttu-id="7a546-109">通常情况下，映像派生自重叠在一起以形成容器的文件系统的层的多个基映像。</span><span class="sxs-lookup"><span data-stu-id="7a546-109">Usually, an image derives from multiple base images that are layers stacked on top of each other to form the container’s filesystem.</span></span> <span data-ttu-id="7a546-110">创建完后，映像是不可变。</span><span class="sxs-lookup"><span data-stu-id="7a546-110">An image is immutable once it has been created.</span></span>

<span data-ttu-id="7a546-111">**容器**: Docker 映像的实例。</span><span class="sxs-lookup"><span data-stu-id="7a546-111">**Container**: An instance of a Docker image.</span></span> <span data-ttu-id="7a546-112">容器表示单个应用程序、 进程或服务的执行。</span><span class="sxs-lookup"><span data-stu-id="7a546-112">A container represents the execution of a single application, process, or service.</span></span> <span data-ttu-id="7a546-113">它包含的内容的 Docker 映像、 执行环境和一组标准的说明。</span><span class="sxs-lookup"><span data-stu-id="7a546-113">It consists of the contents of a Docker image, an execution environment, and a standard set of instructions.</span></span> <span data-ttu-id="7a546-114">缩放服务，当你从相同的映像创建多个容器实例。</span><span class="sxs-lookup"><span data-stu-id="7a546-114">When scaling a service, you create multiple instances of a container from the same image.</span></span> <span data-ttu-id="7a546-115">或者批处理作业可以从同一个映像，将不同的参数传递给每个实例创建多个容器。</span><span class="sxs-lookup"><span data-stu-id="7a546-115">Or a batch job can create multiple containers from the same image, passing different parameters to each instance.</span></span>

<span data-ttu-id="7a546-116">**标记**： 标记或标签，以便可以确定不同的映像或相同的映像 （具体取决于的版本号或目标环境） 的版本，可应用于映像。</span><span class="sxs-lookup"><span data-stu-id="7a546-116">**Tag**: A mark or label you can apply to images so that different images or versions of the same image (depending on the version number or the target environment) can be identified.</span></span>

<span data-ttu-id="7a546-117">**Dockerfile**： 文本文件，其中包含有关如何生成的 Docker 映像的说明。</span><span class="sxs-lookup"><span data-stu-id="7a546-117">**Dockerfile**: A text file that contains instructions for how to build a Docker image.</span></span>

<span data-ttu-id="7a546-118">**生成**： 上的信息和上下文提供其 Dockerfile，以及在其中生成映像的文件夹中的其他文件基于生成的容器映像的操作。</span><span class="sxs-lookup"><span data-stu-id="7a546-118">**Build**: The action of building a container image based on the information and context provided by its Dockerfile, plus additional files in the folder where the image is built.</span></span> <span data-ttu-id="7a546-119">你可以生成使用 Docker docker 生成命令的图像。</span><span class="sxs-lookup"><span data-stu-id="7a546-119">You can build images with the Docker docker build command.</span></span>

<span data-ttu-id="7a546-120">**存储库 （存储库）**： 相关的 Docker 映像，具有标记，该值指示映像版本标记为的集合。</span><span class="sxs-lookup"><span data-stu-id="7a546-120">**Repository (repo)**: A collection of related Docker images, labeled with a tag that indicates the image version.</span></span> <span data-ttu-id="7a546-121">某些存储库包含多个不同版本的一个特定的映像，例如包含映像包含唯一运行时 （较亮） 的 Sdk （更大） 的映像，等等。这些变体可以使用标记进行标记。</span><span class="sxs-lookup"><span data-stu-id="7a546-121">Some repos contain multiple variants of a specific image, such as an image containing SDKs (heavier), an image containing only runtimes (lighter), etc. Those variants can be marked with tags.</span></span> <span data-ttu-id="7a546-122">单个存储库可以包含平台的不同版本，如 Linux 映像和 Windows 映像。</span><span class="sxs-lookup"><span data-stu-id="7a546-122">A single repo can contain platform variants, such as a Linux image and a Windows image.</span></span>

<span data-ttu-id="7a546-123">**注册表**： 提供对存储库的访问的服务。</span><span class="sxs-lookup"><span data-stu-id="7a546-123">**Registry**: A service that provides access to repositories.</span></span> <span data-ttu-id="7a546-124">大多数公共映像的默认注册表是[Docker Hub](https://hub.docker.com/) （归作为组织的 Docker）。</span><span class="sxs-lookup"><span data-stu-id="7a546-124">The default registry for most public images is [Docker Hub](https://hub.docker.com/) (owned by Docker as an organization).</span></span> <span data-ttu-id="7a546-125">注册表通常包含多个团队从存储库。</span><span class="sxs-lookup"><span data-stu-id="7a546-125">A registry usually contains repositories from multiple teams.</span></span> <span data-ttu-id="7a546-126">通常，公司具有私有注册表来存储和管理他们创建的映像。</span><span class="sxs-lookup"><span data-stu-id="7a546-126">Companies often have private registries to store and manage images they’ve created.</span></span> <span data-ttu-id="7a546-127">Azure 容器注册表中是另一个示例。</span><span class="sxs-lookup"><span data-stu-id="7a546-127">Azure Container Registry is another example.</span></span>

<span data-ttu-id="7a546-128">**Docker Hub**： 公共注册表将图像上载和使用它们。</span><span class="sxs-lookup"><span data-stu-id="7a546-128">**Docker Hub**: A public registry to upload images and work with them.</span></span> <span data-ttu-id="7a546-129">Docker Hub 提供 Docker 映像托管、 公共或私有注册表，生成触发器和 web 挂钩，以及与 GitHub 和 Bitbucket 集成。</span><span class="sxs-lookup"><span data-stu-id="7a546-129">Docker Hub provides Docker image hosting, public or private registries, build triggers and web hooks, and integration with GitHub and Bitbucket.</span></span>

<span data-ttu-id="7a546-130">**Azure 容器注册表**： 使用 Docker 映像和在 Azure 中的及其组件的公共资源。</span><span class="sxs-lookup"><span data-stu-id="7a546-130">**Azure Container Registry**: A public resource for working with Docker images and its components in Azure.</span></span> <span data-ttu-id="7a546-131">这提供了的注册表，即将在 Azure 中部署和，它将授予控制访问权限，使其可以使用你的 Azure Active Directory 组和权限。</span><span class="sxs-lookup"><span data-stu-id="7a546-131">This provides a registry that is close to your deployments in Azure and that gives you control over access, making it possible to use your Azure Active Directory groups and permissions.</span></span>

<span data-ttu-id="7a546-132">**Docker 受信任注册表 (DTR)**： 可以是的 Docker 注册表服务 （从 Docker) 安装在本地，因此它存在于组织的数据中心和网络。</span><span class="sxs-lookup"><span data-stu-id="7a546-132">**Docker Trusted Registry (DTR)**: A Docker registry service (from Docker) that can be installed on-premises so it lives within the organization’s datacenter and network.</span></span> <span data-ttu-id="7a546-133">这将为应在企业内管理的专用映像方便。</span><span class="sxs-lookup"><span data-stu-id="7a546-133">It is convenient for private images that should be managed within the enterprise.</span></span> <span data-ttu-id="7a546-134">Docker 受信任注册表是 Docker 数据中心产品的一部分包括。</span><span class="sxs-lookup"><span data-stu-id="7a546-134">Docker Trusted Registry is included as part of the Docker Datacenter product.</span></span> <span data-ttu-id="7a546-135">有关详细信息，请参阅[Docker 受信任注册表 (DTR)](https://docs.docker.com/docker-trusted-registry/overview/)。</span><span class="sxs-lookup"><span data-stu-id="7a546-135">For more information, see [Docker Trusted Registry (DTR)](https://docs.docker.com/docker-trusted-registry/overview/).</span></span>

<span data-ttu-id="7a546-136">**Docker 社区版 (CE)**： 适用于 Windows 和 macOS 有关生成、 运行和测试容器本地的开发工具。</span><span class="sxs-lookup"><span data-stu-id="7a546-136">**Docker Community Edition (CE)**: Development tools for Windows and macOS for building, running, and testing containers locally.</span></span> <span data-ttu-id="7a546-137">CE 用于 Windows 的 docker 提供适用于 Linux 和 Windows 容器的开发环境。</span><span class="sxs-lookup"><span data-stu-id="7a546-137">Docker CE for Windows provides development environments for both Linux and Windows Containers.</span></span> <span data-ttu-id="7a546-138">在 Windows 上的 Linux Docker 主机基于[HYPER-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx)虚拟机。</span><span class="sxs-lookup"><span data-stu-id="7a546-138">The Linux Docker host on Windows is based on a [Hyper-V](https://www.microsoft.com/en-us/server-cloud/solutions/virtualization.aspx) virtual machine.</span></span> <span data-ttu-id="7a546-139">为 Windows 容器主机直接基于 Windows。</span><span class="sxs-lookup"><span data-stu-id="7a546-139">The host for Windows Containers is directly based on Windows.</span></span> <span data-ttu-id="7a546-140">适用于 Mac 的 docker CE 基于 Apple 虚拟机监控程序 framework 和[xhyve 虚拟机监控程序](https://github.com/mist64/xhyve)，提供了 Linux Docker 主机虚拟机上 Mac OS X.CE 用于 Windows 的 Docker 和适用于 Mac 替换 Docker 工具箱中，基于 OracleVirtualBox。</span><span class="sxs-lookup"><span data-stu-id="7a546-140">Docker CE for Mac is based on the Apple Hypervisor framework and the [xhyve hypervisor](https://github.com/mist64/xhyve), which provides a Linux Docker host virtual machine on Mac OS X. Docker CE for Windows and for Mac replaces Docker Toolbox, which was based on Oracle VirtualBox.</span></span>

<span data-ttu-id="7a546-141">**Docker Enterprise Edition (EE)**： 适用于 Linux 和 Windows 开发的 Docker 工具的企业级版本。</span><span class="sxs-lookup"><span data-stu-id="7a546-141">**Docker Enterprise Edition (EE)**: An enterprise-scale version of Docker tools for Linux and Windows development.</span></span>

<span data-ttu-id="7a546-142">**撰写**： 命令行工具和 YAML 文件元数据的定义和运行多容器应用程序的格式。</span><span class="sxs-lookup"><span data-stu-id="7a546-142">**Compose**: A command-line tool and YAML file format with metadata for defining and running multi-container applications.</span></span> <span data-ttu-id="7a546-143">你可以定义单个应用程序基于与一个或多个可以重写值根据环境的.yml 文件的多个映像。</span><span class="sxs-lookup"><span data-stu-id="7a546-143">You define a single application based on multiple images with one or more .yml files that can override values depending on the environment.</span></span> <span data-ttu-id="7a546-144">你已创建定义之后，你可以部署整个多容器应用程序使用单个命令 (docker-向上撰写) 在 Docker 主机上创建每个图像的容器。</span><span class="sxs-lookup"><span data-stu-id="7a546-144">After you have created the definitions, you can deploy the whole multi-container application with a single command (docker-compose up) that creates a container per image on the Docker host.</span></span>

<span data-ttu-id="7a546-145">**群集**： 集合 Docker 公开的主机就像它是一台虚拟 Docker 主机，以便应用程序可以扩展到服务的多个实例分布在群集中的多个主机。</span><span class="sxs-lookup"><span data-stu-id="7a546-145">**Cluster**: A collection of Docker hosts exposed as if it were a single virtual Docker host, so that the application can scale to multiple instances of the services spread across multiple hosts within the cluster.</span></span> <span data-ttu-id="7a546-146">使用 Docker Swarm、 Mesosphere DC/OS、 Kubernetes 和 Azure Service Fabric 可用于创建 docker 群集。</span><span class="sxs-lookup"><span data-stu-id="7a546-146">Docker clusters can be created with Docker Swarm, Mesosphere DC/OS, Kubernetes, and Azure Service Fabric.</span></span> <span data-ttu-id="7a546-147">(如果使用 Docker Swarm 进行管理的群集，你通常是指将群集作为*swarm*而不是群集。)</span><span class="sxs-lookup"><span data-stu-id="7a546-147">(If you use Docker Swarm for managing a cluster, you typically refer to the cluster as a *swarm* instead of a cluster.)</span></span>
