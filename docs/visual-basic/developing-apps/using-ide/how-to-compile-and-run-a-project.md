# <a name="how-to-compile-and-run-a-project-in-visual-basic"></a>如何：在 Visual Basic 中编译和运行项目
[!INCLUDE[vsprvs](../../../csharp/includes/vsprvs_md.md)]集成开发环境 (IDE) 变得简单编译项目并运行生成的应用程序。  
  
 你可以使用 IDE 中内置的调试器在项目运行时对其进行调试。 有关调试的详细信息，请参阅[调试您的 Visual Basic 应用程序](../../../visual-basic/developing-apps/debugging.md)和[编辑并继续 (Visual Basic 中)](https://docs.microsoft.com/visualstudio/debugger/edit-and-continue-visual-basic)。  

[!INCLUDE[note_settings_general](../../../csharp/language-reference/compiler-messages/includes/note_settings_general_md.md)]  
  
### <a name="to-compile-and-run-the-current-project"></a>编译和运行当前项目  
  
-   在 [!INCLUDE[vsprvs](../../../csharp/includes/vsprvs_md.md)] 集成开发环境 (IDE) 中，请按 F5。  
  
     IDE 将编译该项目并在 [!INCLUDE[vsprvs](../../../csharp/includes/vsprvs_md.md)] 调试器中运行应用程序。  
  
### <a name="to-compile-and-run-the-current-project-from-the-menu"></a>从菜单中编译并运行当前项目  
  
-   从**调试**菜单[!INCLUDE[vsprvs](../../../csharp/includes/vsprvs_md.md)]IDE 中，选择**开始调试**。  
  
     IDE 将编译该项目并在 [!INCLUDE[vsprvs](../../../csharp/includes/vsprvs_md.md)] 调试器中运行应用程序。  
  
### <a name="to-compile-and-run-the-current-project-without-debugging"></a>编译并运行当前项目而不调试  
  
-   在 [!INCLUDE[vsprvs](../../../csharp/includes/vsprvs_md.md)] IDE 中按 Ctrl+F5。  
  
     IDE 编译该项目并运行该应用程序。  
  
 璝惠编译并运行[!INCLUDE[vbprvb](../../../csharp/programming-guide/concepts/linq/includes/vbprvb_md.md)]代码使用命令行编译器，请参阅[从命令行生成](../../../visual-basic/reference/command-line-compiler/building-from-the-command-line.md)。
