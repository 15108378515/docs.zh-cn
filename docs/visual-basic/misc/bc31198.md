# <a name="xml-literal-cannot-appear-here-unless-it-is-enclosed-in-parentheses"></a><span data-ttu-id="010db-102">除非括在括号中，否则 XML 文本不能显示在此处</span><span class="sxs-lookup"><span data-stu-id="010db-102">XML literal cannot appear here unless it is enclosed in parentheses</span></span>
<span data-ttu-id="010db-103">在对于 [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] 编译器不明确的位置处，在表达式中使用了 XML 文本声明。</span><span class="sxs-lookup"><span data-stu-id="010db-103">An XML literal declaration is used in an expression in a location that is ambiguous to the [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] compiler.</span></span> <span data-ttu-id="010db-104">也就是说，[!INCLUDE[vbprvb](~/includes/vbprvb-md.md)]编译器无法确定是否小于-不是字符 (<) 旨在作为比较运算符还是 XML 文本的开头。</span><span class="sxs-lookup"><span data-stu-id="010db-104">That is, the [!INCLUDE[vbprvb](~/includes/vbprvb-md.md)] compiler cannot determine whether the less-than character (<) is intended as a comparative operator or the start of an XML literal.</span></span> <span data-ttu-id="010db-105">以下代码提供了一个示例：</span><span class="sxs-lookup"><span data-stu-id="010db-105">The following code provides an example:</span></span>  

```vb  
' Generates an error.

                  Where <sample>Value</sample> = "Value" _  
                  Select element  
```
