# <a name="data-types-of-the-type-parameters-in-extension-method-39ltmethodnamegt39-defined-in-39lttypenamegt39-cannot-be-inferred-from-these-arguments-because-more-than-one-type-is-possible"></a><span data-ttu-id="3e37b-102">扩展方法中类型形参的数据类型&lt;methodname&gt;中定义&lt;typename&gt;因为可能有多个类型不能从这些实参推断</span><span class="sxs-lookup"><span data-stu-id="3e37b-102">Data type(s) of the type parameter(s) in extension method &#39;&lt;methodname&gt;&#39; defined in &#39;&lt;typename&gt;&#39; cannot be inferred from these arguments because more than one type is possible</span></span>
<span data-ttu-id="3e37b-103">扩展方法中类型形参的数据类型\<methodname&1;> 中定义\<typename&1;> 因为可能有多个类型不能从这些实参推断。</span><span class="sxs-lookup"><span data-stu-id="3e37b-103">Data type(s) of the type parameter(s) in extension method '\<methodname>' defined in '\<typename>' cannot be inferred from these arguments because more than one type is possible.</span></span> <span data-ttu-id="3e37b-104">显式指定数据类型可更正此错误。</span><span class="sxs-lookup"><span data-stu-id="3e37b-104">Specifying the data type(s) explicitly might correct this error.</span></span>  
  
 <span data-ttu-id="3e37b-105">尝试使用类型推理功能在对泛型扩展方法的调用中确定类型参数或参数的类型。</span><span class="sxs-lookup"><span data-stu-id="3e37b-105">An attempt has been made to use type inference to determine the type (or types) of the type parameter (or parameters) in a call to a generic extension method.</span></span> <span data-ttu-id="3e37b-106">编译器发现一个或多个类型参数的多个可能的数据类型，并报告此错误。</span><span class="sxs-lookup"><span data-stu-id="3e37b-106">The compiler finds more than one possible data type for one or more of the type parameters, and it reports this error.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="3e37b-107">当无法指定实参时（例如，对于查询表达式中的查询运算符），显示的错误消息不包括第二个句子。</span><span class="sxs-lookup"><span data-stu-id="3e37b-107">When specifying arguments is not an option (for example, for query operators in query expressions), the error message appears without the second sentence.</span></span>  
  
 <span data-ttu-id="3e37b-108">下面的代码演示了此错误。</span><span class="sxs-lookup"><span data-stu-id="3e37b-108">The following code demonstrates the error.</span></span>  
  
```vb  
Option Strict Off  
Imports System.Runtime.CompilerServices  
Module Module1  
    Sub Main()  
  
        Dim caller As New Class1  
        '' Not valid.  
        'caller.targetExtension(1, "2")  
  
    End Sub  
  
    <Extension()> _  
    Sub targetExtension(Of T)(ByVal p0 As Class1, ByVal p1 As T, ByVal p2 As T)  
    End Sub  
  
    Class Class1  
    End Class  
  
End Module  
```

