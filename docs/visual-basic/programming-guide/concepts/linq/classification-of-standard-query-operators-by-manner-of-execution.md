# <a name="classification-of-standard-query-operators-by-manner-of-execution-visual-basic"></a><span data-ttu-id="aaed9-102">标准查询运算符按执行 (Visual Basic 中) 的方式的分类</span><span class="sxs-lookup"><span data-stu-id="aaed9-102">Classification of Standard Query Operators by Manner of Execution (Visual Basic)</span></span>
<span data-ttu-id="aaed9-103">LINQ to 的标准查询运算符方法的对象实现两种主要方法之一执行︰ 立即执行和延迟。</span><span class="sxs-lookup"><span data-stu-id="aaed9-103">The LINQ to Objects implementations of the standard query operator methods execute in one of two main ways: immediate or deferred.</span></span> <span data-ttu-id="aaed9-104">使用延迟的执行的查询运算符可以进一步分为两个类别︰ 流式处理和非流式处理。</span><span class="sxs-lookup"><span data-stu-id="aaed9-104">The query operators that use deferred execution can be additionally divided into two categories: streaming and non-streaming.</span></span> <span data-ttu-id="aaed9-105">如果您知道如何执行不同的查询运算符，它可帮助你了解从给定的查询获得的结果。</span><span class="sxs-lookup"><span data-stu-id="aaed9-105">If you know how the different query operators execute, it may help you understand the results that you get from a given query.</span></span> <span data-ttu-id="aaed9-106">如果更改数据源，或如果您正在构建在另一个查询之上的查询，也是如此。</span><span class="sxs-lookup"><span data-stu-id="aaed9-106">This is especially true if the data source is changing or if you are building a query on top of another query.</span></span> <span data-ttu-id="aaed9-107">本主题对标准查询运算符的执行方式对其进行分类。</span><span class="sxs-lookup"><span data-stu-id="aaed9-107">This topic classifies the standard query operators according to their manner of execution.</span></span>  
  
## <a name="manners-of-execution"></a><span data-ttu-id="aaed9-108">执行方式</span><span class="sxs-lookup"><span data-stu-id="aaed9-108">Manners of Execution</span></span>  
  
### <a name="immediate"></a><span data-ttu-id="aaed9-109">即时</span><span class="sxs-lookup"><span data-stu-id="aaed9-109">Immediate</span></span>  
 <span data-ttu-id="aaed9-110">立即执行意味着数据源读取和执行此操作是在代码中的点查询声明的位置。</span><span class="sxs-lookup"><span data-stu-id="aaed9-110">Immediate execution means that the data source is read and the operation is performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="aaed9-111">返回一个非可枚举的结果的所有标准查询运算符查询立即执行。</span><span class="sxs-lookup"><span data-stu-id="aaed9-111">All the standard query operators that return a single, non-enumerable result execute immediately.</span></span>  
  
### <a name="deferred"></a><span data-ttu-id="aaed9-112">已推迟</span><span class="sxs-lookup"><span data-stu-id="aaed9-112">Deferred</span></span>  
 <span data-ttu-id="aaed9-113">延迟的执行意味着执行的操作是不在代码中的点查询声明的位置。</span><span class="sxs-lookup"><span data-stu-id="aaed9-113">Deferred execution means that the operation is not performed at the point in the code where the query is declared.</span></span> <span data-ttu-id="aaed9-114">仅当枚举查询变量时，例如通过使用时，才执行此操作`For Each`语句。</span><span class="sxs-lookup"><span data-stu-id="aaed9-114">The operation is performed only when the query variable is enumerated, for example by using a `For Each` statement.</span></span> <span data-ttu-id="aaed9-115">这意味着，执行查询的结果取决于数据源的内容而不是当定义查询执行查询时。</span><span class="sxs-lookup"><span data-stu-id="aaed9-115">This means that the results of executing the query depend on the contents of the data source when the query is executed rather than when the query is defined.</span></span> <span data-ttu-id="aaed9-116">如果多次枚举查询变量时，结果可能有所不同每次。</span><span class="sxs-lookup"><span data-stu-id="aaed9-116">If the query variable is enumerated multiple times, the results might differ every time.</span></span> <span data-ttu-id="aaed9-117">其返回类型是几乎所有标准查询运算符<xref:System.Collections.Generic.IEnumerable%601>或<xref:System.Linq.IOrderedEnumerable%601>采用延迟方式执行。</xref:System.Linq.IOrderedEnumerable%601> </xref:System.Collections.Generic.IEnumerable%601></span><span class="sxs-lookup"><span data-stu-id="aaed9-117">Almost all the standard query operators whose return type is <xref:System.Collections.Generic.IEnumerable%601> or <xref:System.Linq.IOrderedEnumerable%601> execute in a deferred manner.</span></span>  
  
 <span data-ttu-id="aaed9-118">使用延迟的执行的查询运算符可以另外分类为流式和非流式处理。</span><span class="sxs-lookup"><span data-stu-id="aaed9-118">Query operators that use deferred execution can be additionally classified as streaming or non-streaming.</span></span>  
  
#### <a name="streaming"></a><span data-ttu-id="aaed9-119">流式处理</span><span class="sxs-lookup"><span data-stu-id="aaed9-119">Streaming</span></span>  
 <span data-ttu-id="aaed9-120">流式运算符不需要在生成元素前读取所有源数据。</span><span class="sxs-lookup"><span data-stu-id="aaed9-120">Streaming operators do not have to read all the source data before they yield elements.</span></span> <span data-ttu-id="aaed9-121">执行时，流运算符执行的操作对每个源元素，因为它读取，并生成该元素，如果合适。</span><span class="sxs-lookup"><span data-stu-id="aaed9-121">At the time of execution, a streaming operator performs its operation on each source element as it is read and yields the element if appropriate.</span></span> <span data-ttu-id="aaed9-122">流式运算符将持续读取源元素，直到可以生成一个结果元素。</span><span class="sxs-lookup"><span data-stu-id="aaed9-122">A streaming operator continues to read source elements until a result element can be produced.</span></span> <span data-ttu-id="aaed9-123">这意味着可能会读取多个源元素生成一个结果元素。</span><span class="sxs-lookup"><span data-stu-id="aaed9-123">This means that more than one source element might be read to produce one result element.</span></span>  
  
#### <a name="non-streaming"></a><span data-ttu-id="aaed9-124">非流式处理</span><span class="sxs-lookup"><span data-stu-id="aaed9-124">Non-Streaming</span></span>  
 <span data-ttu-id="aaed9-125">非流式运算符必须读取所有源数据，它们可以产生结果元素之前。</span><span class="sxs-lookup"><span data-stu-id="aaed9-125">Non-streaming operators must read all the source data before they can yield a result element.</span></span> <span data-ttu-id="aaed9-126">操作，如排序或分组归入此类别。</span><span class="sxs-lookup"><span data-stu-id="aaed9-126">Operations such as sorting or grouping fall into this category.</span></span> <span data-ttu-id="aaed9-127">在执行时，非流式查询运算符读取所有源数据、 将其放到数据结构、 执行操作，然后生成结果元素。</span><span class="sxs-lookup"><span data-stu-id="aaed9-127">At the time of execution, non-streaming query operators read all the source data, put it into a data structure, perform the operation, and yield the resulting elements.</span></span>
